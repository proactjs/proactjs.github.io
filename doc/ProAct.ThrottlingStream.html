Class: ThrottlingStream

ProAct. ThrottlingStream

new ThrottlingStream(queueName, source, transforms, delay)

Constructs a ProAct.ThrottlingStream. This is special kind of ProAct.DelayedStream.

The main idea is the following : if n values/events are triggered to this stream before the time delay for flushing passes, only the last one, the n-th is emitted.

ProAct.ThrottlingStream is part of the streams module of ProAct.js.

Parameters:
Name Type Description
queueName String The name of the queue all the updates should be pushed to.

If this parameter is null/undefined the default queue of ProAct.flow is used.

If this parameter is not a string it is used as the source.

source ProAct.Actor A default source of the stream, can be null.

If this is the only one passed argument and it is a number - it becomes the delay of the stream.

transforms Array A list of transformation to be used on all incoming chages.

If the arguments passed are two and this is a number - it becomes the delay of the stream.

delay Number The time delay to be used to flush the stream.

trigger(event, useTransformations) → {ProAct.ThrottlingStream}

Triggers a new event/value to the stream. It is stored in the buffer of the stream and not emitted. But the buffer of ProAct.DebouncingStream can store only one value/event, so when the delay passes only the last value/event triggered into the stream by this method is emitted. On every call of this method the delay is reset. So for example if you have mouse move as source, it will emit only the last mouse move event, that was send delay milliseconds ago.

ProAct.DebouncingStream.t is alias of this method.

Parameters:
Name Type Description
event Object The event/value to pass to trigger.
useTransformations Boolean If the stream should transform the triggered value. By default it is true (if not passed)

willUpdate(source, actions, eventData) → {ProAct.Actor}

willUpdate() is the method used to notify observers that this ProAct.Actor will be updated.

It uses the ProAct.Actor#defer to defer the listeners of the listening ProAct.Actors. The idea is that everything should be executed in a running ProAct.Flow, so there will be no repetative updates.

The update value will come from the ProAct.Actor#makeEvent method and the source parameter will be passed to it.

If this ProAct.Actor has a parent ProAct.Actor it will be notified in the running flow as well.
