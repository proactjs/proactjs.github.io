Class: AutoProperty

ProAct. AutoProperty

new AutoProperty(proObject, property)

Constructs a ProAct.AutoProperty. The properties are simple ProAct.Actors with state. The auto-computed or functional property has a state of a Function value.

Auto-computed properties are functions which are turned to ProAct.Propertys by a ProAct.ObjectCore.

If these functions are reading another fields of ProAct.js objects, they authomatically become dependent on them.

For example: 

   var obj = {
     a: 1,
     b: 2,
     c: function () {
       return this.a - this.b;
     }
   };
If this object - obj is turned to a reactive ProAct.js object, it becomes a simple object with three fields: 
   {
     a: 1,
     b: 2,
     c: -1
   }
But now c is dependent on a and b, so if a is set to 4, obj becomes: 
   {
     a: 1,
     b: 2,
     c: 2
   }

The logic is the following:

  • The property is initialized to be lazy, so its state is ProAct.States.init
  • On its first read, the ProAct.currentCaller is set to the listener of the property, so all the properties read in the function body became observed by it. The value of the property is computed using the original function of the field.
  • On this first read the state of the property is updated to ProAct.States.ready.
  • On its following reads it is a simple value, computed from the first read. No re-computations on get.
  • If a property, this auto-computed property depends changes, the value of this ProAct.AutoProperty is recomputed.
  • Setting the property can be implemented easy, because on set, the original function of the property is called with the new value.

ProAct.AutoProperty can be depend on another ProAct.AutoProperty.

ProAct.AutoProperty is part of the properties module of ProAct.js.

willUpdate(source, actions, eventData) → {ProAct.Actor}

willUpdate() is the method used to notify observers that this ProAct.Actor will be updated.

It uses the ProAct.Actor#defer to defer the listeners of the listening ProAct.Actors. The idea is that everything should be executed in a running ProAct.Flow, so there will be no repetative updates.

The update value will come from the ProAct.Actor#makeEvent method and the source parameter will be passed to it.

If this ProAct.Actor has a parent ProAct.Actor it will be notified in the running flow as well.

Parameters:
Name Type Description
source Object The source of the update, for example update of ProAct.Actor, that this is observing.

Can be null - no source.

In the most cases ProAct.Event is the source.

actions Array | String A list of actions or a single action to update the listeners that listen to it. If there is no action provided, the actions from ProAct.Actor#defaultActions are used.
eventData Array Data to be passed to the event to be created.
